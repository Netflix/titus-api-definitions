Image

In the job submit operation one of these values should be specified only, ther other will be resolved by Titus. The resulting job descriptor will include always both values. For the deployment purposes only 'imageId' value is used. The resolution happens only once, immediately during the submit type, so the same image is used for all tasks. This behavior is not applied for job cluster. This means that if a name contains 'latest' version, each job created from the same cluster definition, may have different effective image version.

FieldTypeLabelDescription

Maximum amount of time in seconds that the job is allowed to run. Time is measured since the job is submitted.

Mantis job can be auto-terminated if there is no subscription to it. To prevent early terminations, a minimum running time can be specified via this property.

For 'Perpetual' type, do not terminate idle jobs. For 'Transient' jobs, the early termination is on, and is controlled by 'minRuntimeSecs' property.

Mantis specific. Users can provide a value to dedup job submissions. Users can “submit-and-connect” to jobs such that only one job per unique value of this type actually is created. Other requests connect to the existing job instead of creating another one.

TaskInstances

This data structure is associated with a job, and specifices number of tasks to run (desired). If job supports auto-scaling (service and Mantis jobs), min/max define constraints. At any point in time the condition min <= desired <= max must hold true. The desired state may be changed a user, but also may be changed as a result of auto-scaling action.

Minimum number of tasks to run. If not set, defaults to 0.
max uint32 optional

Maximum number of tasks that can be run. If not set, set to 2^31-1 (Integer.MAX_VALUE).
